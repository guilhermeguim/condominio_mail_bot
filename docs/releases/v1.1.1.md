## Objective

Fix the user-facing timestamp shown in the Telegram success message so it reflects Brasilia time rather than the container runtime's default timezone.

## Release Procedures Executed

- Confirmed that the timezone hotfix was merged into `main` before preparing the patch release notes.
- Collected the diff statistics and merged pull requests included since `v1.1.0`.
- Updated `CHANGELOG.md` with the `v1.1.1` entry.
- Added this release dossier under `docs/releases/`.

## Main Deliveries

### Timestamp correction

- Replaced the naive success-message timestamp with an explicit Brasilia timezone.
- Added a fallback to UTC-3 when `ZoneInfo` data is not available in the runtime.

### User feedback consistency

- Kept the Telegram confirmation message aligned with the user's expected local time.
- Reduced ambiguity for operators validating the end-to-end delivery flow.
