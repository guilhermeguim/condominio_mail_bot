## Objective

Move the outbound email subject and body into environment configuration so the message copy can be changed without editing the source code.

## Release Procedures Executed

- Confirmed that the environment-configuration refactor was merged into `main` before preparing the patch release notes.
- Collected the diff statistics and merged pull requests included since `v1.1.1`.

- Added this release dossier under `docs/releases/`.

## Main Deliveries

### Email copy externalization

- Removed the hardcoded outbound email subject from the Microsoft Graph payload builder.
- Removed the hardcoded outbound email body from the email delivery module.
- Loaded `EMAIL_SUBJECT` and `EMAIL_BODY` from environment configuration at send time.

### Runtime formatting behavior

- Converted escaped `\n` sequences from `EMAIL_BODY` into actual line breaks before sending the message.
- Kept the message formatting configurable without re-deploying code changes for copy-only adjustments.

### Documentation updates

- Updated the README environment example to include `EMAIL_SUBJECT` and `EMAIL_BODY`.
- Updated the Cloud Run deployment example to include the new required variables.

## Compatibility

- Python runtime target: 3.11+
- Existing deployments must now define `EMAIL_SUBJECT` and `EMAIL_BODY` in addition to the previously required environment variables.
- `EMAIL_BODY` may include escaped `\n` sequences when a multi-line email message is required.

## Recommended Deployment Procedure

1. Build the new image from the repository root.
2. Publish the image to the configured container registry.
3. Deploy the image to Cloud Run with `EMAIL_SUBJECT` and `EMAIL_BODY` included in the runtime environment.
4. Confirm the Telegram webhook still points to the final `/webhook` endpoint.

Reference operational guide: `docs/operations.md`.

## Post-Deploy Smoke Test

1. Confirm the new Cloud Run revision starts successfully.
2. Send a valid PDF from an authorized Telegram chat.
3. Verify the delivered email uses the configured subject from `EMAIL_SUBJECT`.
4. Verify the delivered email body matches `EMAIL_BODY`, including expected line breaks.

## Rollback

- Roll back to `v1.1.1` if the runtime configuration for the new email variables is missing or behaves unexpectedly in production.
- Re-check the Cloud Run environment values before attempting a new rollout.
